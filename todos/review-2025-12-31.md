## 🔴 CRITICAL ISSUES (Must Fix)

### Security
1. **PowerShell Command Injection** - `cmd/commands.go:434-513`
   - User-controlled directory paths interpolated into PowerShell scripts
   - Risk: Arbitrary code execution on Windows

2. **Token Exposure in Logs** - `internal/tui/logger/logger.go:203`
   - Full request/response bodies logged, may contain Bearer tokens
   - Log files at `~/.claude2kiro/logs/*.log` with 0644 permissions

### Performance
3. **O(n²) String Operations** - `internal/tui/logger/logger.go:324-327`
   - Multiple space collapse uses repeated string replacement
   - Can iterate 10-20 times for large bodies

4. **Unbounded HTTP ReadAll** - `main.go:867`
   - `io.ReadAll()` without size limits
   - Risk: OOM from large/malicious responses

### Data Integrity
5. **Non-Atomic Config Save** - `internal/config/config.go:167-185`
   - `os.WriteFile` directly overwrites config
   - Risk: Config corruption on crash

6. **Non-Atomic Token Save** - `main.go:1809, 1882, 2055`
   - Token file writes not atomic
   - Risk: User locked out if crash during write

7. **Unbuffered Log Writes** - `internal/tui/logger/logger.go:304`
   - No error handling, no buffering, no flush
   - Risk: Silent data loss on disk full

### Architecture
8. **main.go is 2,653 lines** - Contains 49 functions, 25 types
    - HTTP server, auth flows, token management, CLI all in one file
    - Should extract to internal/server/, internal/auth/, internal/token/

9. **Type Duplication** - TokenData defined in main.go AND cmd/commands.go
    - Also duplicate: CreditsInfo, TokenInfo, ClaudeConfigStatus, messages

## 🟡 IMPORTANT ISSUES (Should Fix)

### Security
12. **Insecure File Permissions** - `config/config.go:184`
    - Config/temp files use 0644 (world-readable)
    - Should use 0600 for sensitive files

13. **Missing HTTP Security Headers** - `main.go:792-795`
    - CORS allows all origins (`*`)
    - No CSP, X-Frame-Options, X-Content-Type-Options

14. **No Rate Limiting** - Proxy server `/v1/messages`
    - Risk: DoS, credit abuse, brute force

### Performance
15. **O(n) Filter Scan** - Every filter change rescans all entries
    - Should use incremental filtering with indices

16. **Unbuffered File I/O** - Each log entry = 1 syscall
    - Should use buffered writer with periodic flush

17. **No HTTP Connection Pooling** - New clients created inconsistently
    - Should reuse single http.Client with connection pool

### Architecture
18. **Message Type Duplication** - `internal/tui/messages.go` vs `messages/messages.go`
    - Two files define same types (ServerStartedMsg, ServerStoppedMsg)

19. **Circular Dependency Risk** - cmd ↔ dashboard potential cycle
    - Workaround via type aliases, fragile architecture

20. **Global Config State** - `config.Get()` uses unprotected global var
    - Race condition, testing difficulty

### Data Integrity
21. **No Concurrent Write Protection** - config.Save() has no mutex
22. **Silent Parse Errors** - config.Load() swallows YAML errors

### Agent-Native (CLI Gaps)
24. **No `status --json`** - Cannot check if server is running programmatically
25. **No `credits --json`** - Credit info only in TUI
26. **No `logs --json`** - Log access only in TUI
27. **No health endpoint** - No `/health` for orchestration
28. **No graceful shutdown** - Must kill process, no SIGTERM handler
