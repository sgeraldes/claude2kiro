# v1.5.0

Consolidates the credit-visibility / credit-reduction feature line (v1.4.4 —
v1.4.6) and closes its two tracked follow-ups.

## Fixes

- **Concurrent proxies no longer clobber credit history** (#21). Steady-state
  persistence of `~/.claude2kiro/credit-history.jsonl` is now append-only
  (one atomic line per sample) instead of a whole-file rewrite, so a
  persistent `server` and any number of `claude2kiro run` proxies can sample
  side by side without losing each other's points. Each proxy re-reads the
  file after sampling, so every dashboard converges on the union of all
  samples. The file is compacted (stale lines dropped) only once at startup.
- **`tool_mode=compact` no longer splits UTF-8 runes** (#22). Description
  truncation now backs up to a rune boundary, so multi-byte characters are
  never cut in half.

## Feature line summary (since v1.4.3)

- Credit usage history + live burn-rate/projected-runout dashboard
  (reset-aware), per-request Kiro credit metering in logs.
- Kiro protocol parity: `agentTaskType`, native effort, `envState`, stable
  `conversationId` (default ON).
- Opt-in request-diet controls (`history_mode`, `tool_mode`,
  `aggressive_cache_points`) with protocol-aware trimming that preserves
  tool-use/result pairs and the system prompt; local benchmarks measured
  15-68% credit reductions depending on mode.
- Virgin-machine bootstrap: auto-install Claude Code (winget/script/npm) and
  Claude Desktop (winget), non-destructive `~/.claude.json` seeding (no
  Anthropic account needed for the CLI), and Desktop gateway config written
  automatically.

See `docs/releases/v1.4.5.md` and `docs/releases/v1.4.6.md` for details.
