# v1.4.6

Supersedes v1.4.5, which was published from a pre-merge branch commit before
review fixes landed. Delta since the published v1.4.5:

## Virgin-Machine Bootstrap

- `claude2kiro run`/`remote` detect a missing Claude Code CLI and offer to
  install it (winget native build on Windows — no Node.js needed — with
  Anthropic's install script and npm as fallbacks), resolving the fresh
  binary even while the terminal's PATH is stale.
- `~/.claude.json` is seeded non-destructively (onboarding flags, proxy key
  approval, API-key account stub) so Claude Code skips its first-run login
  wizard entirely — the CLI needs no Anthropic account. A real claude.ai
  login is never overwritten; corrupt files are left untouched.
- `claude2kiro desktop` installs Claude Desktop via winget when missing and
  writes the gateway managed config (`Claude-3p\configLibrary`) pointing at
  the proxy — previously nothing created it, so a fresh machine's Desktop
  routed straight to Anthropic. Foreign managed configs abort the launch
  instead of being overwritten. Note: Desktop plan-gates the Code/Cowork
  tabs client-side — a free claude.ai account only unlocks Chat.
- Install prompts decline automatically in non-interactive sessions.

## Credit Feature Fixes (review findings)

- Dashboard burn rate / projected runout no longer dies across the monthly
  Kiro credit reset: it now measures from the most recent reset forward,
  preferring the last 24 hours of samples.
- `history_mode=recent`/`current_only` no longer drop the synthesized
  system-prompt history pairs — the model keeps its instructions on every
  dieted request (regression tests added).
- Removed dead history-diet helper functions that failed the lint gate;
  `setClaude` now reuses the shared non-destructive config seeding.

## Docs

- README covers fresh-machine setup and the Desktop sign-in caveat.
