# v1.4.5

## Virgin-Machine Bootstrap

- `claude2kiro run`/`remote` detect a missing Claude Code CLI and offer to
  install it (winget native build on Windows — no Node.js needed — with
  Anthropic's install script and npm as fallbacks), resolving the fresh
  binary even while the terminal's PATH is stale.
- `~/.claude.json` is now seeded non-destructively (onboarding flags, proxy
  key approval, API-key account stub) so Claude Code skips its first-run
  login wizard entirely — the CLI needs no Anthropic account. A real
  claude.ai login is never overwritten; corrupt files are left untouched.
- `claude2kiro desktop` installs Claude Desktop via winget when missing and
  writes the gateway managed config (`Claude-3p\configLibrary`) pointing at
  the proxy — previously nothing created it, so a fresh machine's Desktop
  routed straight to Anthropic. Foreign managed configs are never overwritten
  (the launch aborts instead). Note: Desktop plan-gates the Code/Cowork tabs
  client-side — a free claude.ai account only unlocks Chat.
- Install prompts decline automatically in non-interactive sessions.

## Credit and Protocol Revamp

- Added current Kiro protocol parity fields: `agentTaskType`, native
  `additionalModelRequestFields.output_config.effort`, and Claude Code `<env>`
  passthrough as `envState`.
- Kept stable session-derived `conversationId` enabled by default to match Kiro's
  pay-per-request session behavior more closely.
- Added Kiro `meteringEvent` parsing and request metrics logging so local runs can
  compare exact credits, request bytes, history size, and tool counts.

## Experimental Credit Controls

- Added TUI settings for experimental request diet controls:
  `history_mode`, `history_recent_turns`, `tool_mode`,
  `tool_compact_max_chars`, and `aggressive_cache_points`.
- Defaults stay conservative: full history, full tools, and aggressive cache
  points disabled.
- History trimming is protocol-aware and preserves required tool-use/tool-result
  pairs to avoid backend `TOOL_USE_RESULT_MISMATCH` errors.

## Testing and Benchmarking

- Added `scripts/benchmark-credit.ps1` for repeatable Claude Code headless
  comparisons across full, current-only, recent-compact, none-text, and
  aggressive-cache modes.
- Added regression coverage for native effort, env state, history diet modes,
  tool diet modes, aggressive cache points, metering parsing, and tool-result
  history pairing.
- Final local validation:
  - `go test ./...`
  - `cmd /c build.bat 1.4.5`
  - `scripts/benchmark-credit.ps1 -Binary .\claude2kiro.exe -Scenario tool`
  - `scripts/benchmark-credit.ps1 -Binary .\claude2kiro.exe -Scenario no-tool`

## Benchmark Results

- No-tool matrix: `none-text` reduced credits by 68.1%; `aggressive-cache`
  reduced credits by 36.0%; `current-only` reduced credits by 14.6%.
- Tool matrix: `aggressive-cache` reduced credits by 33.5%; `current-only`
  reduced credits by 30.5%; `recent-compact` reduced credits by 27.7%.
- Detailed local results are in
  `docs/benchmarks/2026-06-30-credit-reduction.md`.
